Cloudflare hardening

We cannot configure Cloudflare from this repo. Use the dashboard to add the following. All rules use Managed Challenge (not block) so real browsers can pass.

Rule 1: Challenge common bot user agents

Reduces junk traffic and protects the origin from scripted hits.

  • If (http.user_agent contains "curl" OR "python" OR "wget" OR "scrapy" OR "httpclient") AND hostname equals tools.nextstepsbeyond.online
  • Then Managed Challenge

Rule 2: Challenge common exploit paths

These paths do not exist on this site. Challenging them reduces noise and blocks automated probes.

  • If URI path matches one of: /wp-admin, /wp-login.php, /.env, /phpmyadmin, /server-status, /xmlrpc.php
  • Then Managed Challenge
